API Reference

A API do Averba é REST sobre HTTPS, com payloads em JSON. Todos os endpoints retornam UTF-8 e usam ISO-8601 para datas. Versionamento é prefixado na URL (/v1/) e novas versões são introduzidas sem quebrar versões anteriores por pelo menos 12 meses.

Base URL: https://api.averba.io  ·  Sandbox: https://api.sandbox.averba.io

Autenticação

A API usa JWT (Bearer) com refresh token rotativo. O access token expira em 15 minutos; o refresh token em 30 dias. Envie o token no header Authorization: Bearer <jwt>. Para integrações de banco há também suporte a OAuth2 client credentials + mTLS.

Erros & rate-limit

Erros seguem o formato { "error": { "code", "message", "details" } }. Rate-limit padrão: 600 req/min por token; bursts até 100 req em 10s. Headers X-RateLimit-Remaining e X-RateLimit-Reset informam o estado atual.

GET /v1/servidores/{servidor_id}/margem-consignavel

Retorna a margem consignável atualizada do servidor — salário base, valor comprometido, valor disponível e fonte dos dados (folha da prefeitura). Resposta cacheada por 5 minutos em borda; invalidada automaticamente quando há nova averbação.

Path parameters

NomeTipoObrigatórioDescrição
servidor_id integer required Identificador numérico do servidor municipal cadastrado.

Query parameters

NomeTipoObrigatórioDescrição
force_refresh boolean optional Se true, ignora cache e consulta a folha da prefeitura em tempo real. Padrão: false.
include string optional Recursos adicionais — valores aceitos: contratos, historico. Separados por vírgula.

Exemplos

GET /v1/servidores/{servidor_id}

Retorna os dados cadastrais do servidor: nome, matrícula, prefeitura, status, contratos ativos.

Path parameters

NomeTipoObrigatórioDescrição
servidor_idintegerrequiredID do servidor.
GET /v1/servidores

Lista paginada de servidores (apenas para clientes administrativos). Suporta filtros por prefeitura, status e busca textual.

Query parameters

NomeTipoObrigatórioDescrição
prefeitura_idintegeroptionalFiltra por prefeitura.
statusstringoptionalativo · bloqueado · arquivado
cursorstringoptionalCursor de paginação (retornado em meta.next_cursor).
limitintegeroptional1–100. Padrão 25.
POST /v1/propostas/simular

Faz fan-out de simulação para todos os bancos elegíveis e retorna lista ordenada por CET. Latência típica 800–1500ms.

Request body

NomeTipoObrigatórioDescrição
servidor_idintegerrequiredID do servidor solicitante.
valornumberrequiredValor desejado em BRL (mínimo R$ 500).
parcelasintegerrequiredNúmero de parcelas (12 a 96).
bancosarray<int>optionalRestringe simulação a IDs de bancos específicos.
POST /v1/propostas

Cria uma proposta a partir de uma simulação aceita pelo servidor. Dispara webhook para o banco e abre fluxo de assinatura digital.

POST /v1/propostas/portabilidade

Inicia processo de portabilidade de um contrato existente. Workflow assíncrono — retorne ao endpoint de status para acompanhar.

GET /v1/bancos

Lista bancos ativos elegíveis para o servidor autenticado, com produtos, faixas de taxa e status do serviço.

GET /v1/prefeituras

Lista prefeituras afiliadas. Cada item inclui status da última sincronização da folha.

POST /v1/webhooks

Registra um endpoint para receber eventos. Eventos disponíveis: proposta.criada, proposta.aprovada, proposta.contratada, proposta.averbada, portabilidade.concluida.

GET /v1/webhooks/eventos

Lista eventos disparados, com status de entrega, próximas tentativas e payload original (para depuração).
