Integração · API REST + Webhooks

Como o banco se conecta

Integração padronizada e versionada — onboard em até 5 dias úteis. Sandbox completo, autenticação OAuth2 + mTLS, webhooks idempotentes.

Endpoints expostos

O banco implementa 4 endpoints REST: simular, criar proposta, consultar status, cancelar. Contrato OpenAPI 3.1 público.

POST /v1/simular
POST /v1/propostas
GET /v1/propostas/{id}
POST /v1/propostas/{id}/cancelar
+ Mais ›
Spec OpenAPI versionada. Geração automática de SDK em TS/Python. Testes de contrato no pipeline.

Autenticação

OAuth2 client credentials + mTLS opcional. Token cacheado pelo Averba com refresh proativo a 80% do TTL.

POST /oauth/token
grant_type=client_credentials
scope=propostas:rw
→ access_token (TTL 3600s)
+ Mais ›
Credenciais por ambiente (sandbox/prod). Rotação assistida. mTLS validado no perímetro Cloudflare.

Webhooks de callback

Eventos enviados pelo banco quando proposta muda de estado: aprovada, contratada, averbada, cancelada, com payload assinado.

X-Averba-Signature: sha256=...
X-Averba-Event: proposta.aprovada
X-Averba-Idempotency: 9821-aprov
+ Mais ›
HMAC-SHA256 com segredo compartilhado. Retry exponencial (3 tentativas) com dead-letter. Deduplicação por idempotency key (24h).
